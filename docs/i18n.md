# Internationalization (i18n)

VAXPUI provides full support for international text, including Arabic, Chinese, Japanese, Korean, and all Unicode scripts.

## Features

- ✅ Full Unicode support
- ✅ Right-to-Left (RTL) text
- ✅ Arabic text shaping (letter joining)
- ✅ CJK characters
- ✅ Emoji support
- ✅ System input methods (IBus, fcitx)

## Technical Implementation

### XIM (X Input Method)

VAXPUI uses the standard X11 input method protocol:

```
Keyboard → IBus/fcitx → XIM → XIC → Xutf8LookupString → Widget
```

### Pango Text Rendering

Text is rendered using **PangoCairo**, which provides:

- Automatic text shaping (HarfBuzz)
- RTL bidirectional text
- Proper font fallback
- Complex script support

## Using International Text

### Display Text

```c
// Arabic
vaxp_text("مرحباً بالعالم")

// Chinese
vaxp_text("你好世界")

// Japanese
vaxp_text("こんにちは")

// Korean
vaxp_text("안녕하세요")

// Mixed text
vaxp_text("Hello مرحباً 你好")
```

### Text Input

```c
vaxp_input(
    .placeholder = "أدخل النص هنا",  // Arabic placeholder
    .on_change = handle_text
)

void handle_text(VaxpTextInput* input, const char* text, void* data) {
    // 'text' is UTF-8 encoded
    printf("Input: %s\n", text);
}
```

All text input is UTF-8 encoded and properly handles:

- Multi-byte characters
- Backspace deletes full characters (not bytes)
- Cursor movement respects character boundaries

## RTL Support

Arabic and Hebrew text automatically renders right-to-left:

```c
// This text will render right-to-left
vaxp_text("مرحباً بالعالم")

// Output: العالم بمرحباً (displayed right-to-left)
```

## Arabic Text Shaping

Arabic letters connect differently based on position:

| Position | ب Example |
|----------|-----------|
| Isolated | ب |
| Initial | بـ |
| Medial | ـبـ |
| Final | ـب |

VAXPUI handles this automatically via Pango/HarfBuzz.

**Example:**

```c
vaxp_text("سلام")  // Properly shaped as connected letters
// Not: س ل ا م (isolated letters)
```

## UTF-8 Text Handling

All text in VAXPUI is UTF-8:

### String Length

```c
// UTF-8 string - strlen returns bytes, not characters
const char* arabic = "مرحباً";
size_t bytes = strlen(arabic);        // e.g., 12 bytes
// For character count, use specialized function

// Get UTF-8 character length
size_t char_len(const char* s) {
    size_t len = 0;
    while (*s) {
        if ((*s & 0xC0) != 0x80) len++;  // Count leading bytes
        s++;
    }
    return len;
}
```

### Processing UTF-8

```c
// Get next UTF-8 character
const char* next_utf8_char(const char* s, size_t* char_bytes) {
    if (!s || !*s) return NULL;
    
    unsigned char c = (unsigned char)*s;
    if (c < 0x80) *char_bytes = 1;
    else if ((c & 0xE0) == 0xC0) *char_bytes = 2;
    else if ((c & 0xF0) == 0xE0) *char_bytes = 3;
    else if ((c & 0xF8) == 0xF0) *char_bytes = 4;
    else *char_bytes = 1;  // Invalid, skip
    
    return s + *char_bytes;
}
```

## Troubleshooting

### Characters Show as Boxes

**Problem:** Text shows as □□□ (tofu boxes)

**Solution:** Install appropriate fonts:
```bash
sudo apt install fonts-noto-cjk fonts-noto  # CJK + Arabic
```

### Input Method Not Working

**Problem:** Can't type non-ASCII characters

**Solution:**
1. Ensure IBus/fcitx is running
2. Check XIM-related environment variables
3. Add to `.bashrc`:
   ```bash
   export GTK_IM_MODULE=ibus
   export XMODIFIERS=@im=ibus
   ```

### Arabic Letters Disconnected

**Problem:** Arabic shows as isolated letters

**Solution:** This is automatically handled by Pango. Ensure libpango1.0-dev is installed.
